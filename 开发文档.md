# 聚发舟 MultiPost AI 多平台内容发布工具开发文档

## 一、项目名称

项目名称：**聚发舟 MultiPost AI**

项目定位：面向内容创作者的 AI 多平台内容适配与一键发布工具。

“聚发舟”表示将创作者的内容统一汇聚，并像一艘内容分发船一样，将内容高效发布到公众号、知乎、B站、小红书等多个平台。

## 二、比赛需求对照

比赛题目要求：

> 用户在工具中输入内容，可自动适配各平台格式与风格，并支持一键发布（可选模拟发布）。需给出扩展更多平台的架构设计。

本项目覆盖情况如下：

| 比赛要求      | 本项目实现方案                        | 是否覆盖 |
| --------- | ------------------------------ | ---- |
| 用户输入内容    | 支持标题、正文、标签、图片、视频、封面输入          | 已覆盖  |
| 自动适配各平台格式 | 适配公众号、知乎、B站、小红书字段结构            | 已覆盖  |
| 自动适配各平台风格 | 使用 AI 改写 + 平台模板规则              | 已覆盖  |
| 一键发布      | 当前支持模拟发布，后续预留官方接口              | 已覆盖  |
| 可选模拟发布    | 支持模拟登录、模拟发布、发布状态记录             | 已覆盖  |
| 扩展更多平台架构  | 使用 Adapter + Publisher 插件式架构   | 已覆盖  |
| 平台账号接入    | 当前模拟登录，后续预留 OAuth/API Token 接口 | 已覆盖  |
| 图片视频上传    | 支持图文、视频、封面上传                   | 已覆盖  |

因此，本项目已经覆盖比赛核心要求，并进一步增加了 AI 改写、多模型兼容、素材上传、平台账号接入预留等扩展功能。

## 三、项目背景

当前创作者经常需要将同一内容同步发布到公众号、知乎、B站、小红书等平台。但不同平台对内容风格、字段结构和素材形式要求不同。

例如：

* 公众号偏正式长文，适合完整结构化表达；
* 知乎偏问答和分析，适合逻辑推导与经验总结；
* B站偏视频投稿，需要标题、简介、分区、封面和视频；
* 小红书偏图文/短视频种草，强调口语化标题、标签和视觉素材。

手动适配多个平台会造成大量重复劳动。因此，本项目设计“聚发舟 MultiPost AI”，实现一次输入、多平台生成、多平台预览、一键发布。

## 四、项目目标

系统目标如下：

1. 支持用户输入原始标题、正文、标签。
2. 支持上传图片、视频、封面图。
3. 支持 AI 自动改写内容。
4. 支持 OpenAI、Gemini、Qwen、DeepSeek、Ollama、本地大模型等模型。
5. 支持公众号、知乎、B站、小红书四个平台适配。
6. 支持平台内容预览和二次编辑。
7. 支持模拟登录和模拟发布。
8. 预留真实平台账号授权接口。
9. 预留真实一键发布接口。
10. 支持发布记录管理。
11. 支持后续扩展微博、抖音、头条、快手等平台。

## 五、界面资源与图标规范

### 5.1 figures 文件夹要求

项目根目录下需要新建 `figures` 文件夹，用于存放界面所需图标资源。

目录结构如下：

```text
multiplatform/
├── figures/
│   ├── 聚发舟.png
│   ├── 公众号.png
│   ├── 小红书.png
│   ├── b站.png
│   └── 知乎.png
```

### 5.2 图标用途说明

| 文件名     | 用途                     |
| ------- | ---------------------- |
| 聚发舟.png | 系统 Logo，显示在左侧导航栏顶部和登录页 |
| 公众号.png | 微信公众号平台卡片图标            |
| 小红书.png | 小红书平台卡片图标              |
| b站.png  | B站平台卡片图标               |
| 知乎.png  | 知乎平台卡片图标               |

### 5.3 图标设计要求

图标风格应保持统一：

1. 简洁、扁平化、现代化。
2. 背景透明或白底。
3. 尺寸建议为 512×512。
4. 页面中显示尺寸建议为 32×32 或 40×40。
5. 聚发舟 Logo 采用橙色帆船形象，突出品牌识别。
6. 平台图标保持平台识别特征，但整体视觉风格要与系统 UI 协调。


### 14.4 封面上传接口

```text
POST /api/upload/cover
```

### 14.5 模拟登录接口

```text
POST /api/account/mock-login
```

请求示例：

```json
{
  "platform": "bilibili",
  "account_name": "测试创作者账号"
}
```

### 14.6 官方授权预留接口

```text
GET /api/account/oauth-url/{platform}
GET /api/account/oauth-callback/{platform}
```

当前阶段返回：

```json
{
  "code": 501,
  "message": "官方授权接口暂未接入，当前使用模拟登录"
}
```

### 14.9 发布记录查询接口

## 十九、总结

“聚发舟 MultiPost AI” 是一个面向内容创作者的多平台智能内容发布工具。系统以 AI 改写、多平台适配、素材上传、模拟登录和一键发布为核心，能够帮助用户将一份原始内容快速转换为适合公众号、知乎、B站、小红书等平台的发布版本。

本项目不仅满足比赛要求，还通过 `figures` 图标资源规范、平台适配器、发布器接口和模型调用接口，为后续真实平台接入和系统扩展提供了清晰架构。
