# 聚发舟 MultiPost AI 二次问题修复与模块优化方案

## 一、前后端联通与数据持久化修复

### 当前问题

当前系统主要是纯前端模拟，注册账户、发布记录、素材信息、平台账号状态等数据大概率存储在 `localStorage` 或前端内存中。一旦关闭前端进程或刷新状态，部分数据会丢失，未形成真正的前后端联通。

### 修复目标

系统需要改为：

```text
前端页面
 ↓ API 请求
后端 FastAPI 服务
 ↓
SQLite 数据库
```

所有核心数据必须由后端保存，包括：

1. 用户注册信息
2. 登录 token
3. 发布任务
4. 平台适配内容
5. 发布记录
6. 素材文件信息
7. 会员状态
8. 平台账号登录状态

### 必须新增的后端接口

```text
POST /api/auth/register       用户注册
POST /api/auth/login          用户登录
GET  /api/auth/profile        获取当前用户信息

POST /api/content/generate    生成多平台内容
GET  /api/content/{task_id}   查看任务详情

POST /api/upload/image        上传图片
POST /api/upload/video        上传视频
POST /api/upload/cover        上传封面
GET  /api/materials           查看素材库

POST /api/publish             一键发布
GET  /api/records             查看发布记录
GET  /api/records/{id}        查看发布详情

POST /api/account/mock-login  模拟登录平台账号
GET  /api/account/list        查看平台账号状态

GET  /api/settings/llm        获取 AI 设置
POST /api/settings/llm        保存 AI 设置
```

### 数据库必须保存的表

```text
user 用户表
content_task 内容任务表
platform_content 平台内容表
media_file 素材表
publish_record 发布记录表
platform_account 平台账号表
vip_record 会员记录表
ai_setting AI 设置表
```

### 判断是否修复成功

关闭前端和后端后重新启动，仍然可以看到：

1. 已注册账户；
2. 已上传素材；
3. 已生成内容；
4. 已发布记录；
5. 已开通会员状态；
6. 已保存 AI 设置。

## 二、发布记录详情页修复

### 当前问题

发布记录页面只能看到发布状态，无法查看每篇帖子具体发布了什么内容。

### 修复目标

发布记录应支持点击进入详情页，查看每个平台发布的完整内容。

### 推荐页面结构

发布记录列表页：

```text
发布时间 | 任务标题 | 发布平台 | 发布状态 | 操作
2026-05-29 | 如何用AI工具提升大学生学习效率 | 公众号/知乎/B站/小红书 | 成功 | 查看详情
```

详情页展示：

```text
任务标题
原始内容
发布时间
发布模式：mock / official
发布平台数量
发布状态

平台内容详情：
- 微信公众号
  - 标题
  - 正文
  - 标签
  - 图片
  - 封面
  - 发布状态

- 知乎
  - 标题
  - 正文
  - 标签
  - 图片
  - 发布状态

- B站
  - 标题
  - 视频简介
  - 分区
  - 标签
  - 视频
  - 封面
  - 发布状态

- 小红书
  - 标题
  - 正文
  - 标签
  - 图片/视频
  - 发布状态
```

### 后端接口


## 三、账户管理、素材管理、AI 设置模块优化

### 当前问题

这三个页面内容较空，并且与“内容创作”页面功能重复，导致用户感觉页面没有价值。

建议不要删除，而是重新定位：
内容创作页负责“当前任务使用”；
独立页面负责“全局管理和历史维护”。

---

### 3.1 账户管理页面优化

#### 当前问题

只是显示几个平台已登录，功能价值较弱。

#### 修改方向

账户管理页面应改为：**平台账号中心**。

功能包括：

1. 查看各平台账号状态；
2. 模拟登录 / 退出登录；
3. 管理账号名称；
4. 显示授权类型；
5. 显示 token 过期时间；
6. 预留官方授权入口；
7. 显示平台发布权限说明。

页面结构：

```text
平台账号中心

平台卡片：
微信公众号
状态：已登录
授权方式：Mock Token
账号名称：测试公众号账号
授权有效期：2026-12-31
操作：重新登录 / 退出登录 / 官方授权预留

知乎
状态：未登录
操作：模拟登录 / 官方授权预留
```

这样账号管理页就不再只是重复内容创作里的登录状态，而是负责全局账号配置。

---

### 3.2 素材管理页面优化

#### 当前问题

素材管理页只有上传框，内容较空，且与内容创作页素材上传重复。

#### 修改方向

素材管理页面应改为：**全局素材库**。

内容创作页的素材上传只服务当前发布任务；
素材管理页用于长期管理用户上传过的全部素材。

功能包括：

1. 查看全部图片；
2. 查看全部视频；
3. 查看全部封面；
4. 按素材类型筛选；
5. 按上传时间排序；
6. 删除素材；
7. 复制素材链接；
8. 查看素材尺寸、大小、格式；
9. 将素材加入当前创作任务；
10. 显示素材使用次数。

页面结构：

```text
素材库

顶部统计：
图片 12 张 | 视频 3 个 | 封面 5 张 | 总占用 128MB

筛选：
全部 / 图片 / 视频 / 封面

素材网格：
缩略图
文件名
格式
尺寸
大小
上传时间
操作：预览 / 复制链接 / 删除 / 加入创作
```

这样素材管理页就有明确价值。

---

### 3.3 AI 设置页面优化

#### 当前问题

AI 设置只显示模型选择、改写强度、输出语言，与内容创作页重复。

#### 修改方向

AI 设置页面应改为：**全局 AI 配置中心**。

内容创作页只选择本次生成时使用的模型；
AI 设置页用于配置所有模型的 API Key 和默认参数。

功能包括：

1. 设置默认模型；
2. 配置 OpenAI API Key；
3. 配置 Gemini API Key；
4. 配置 Qwen API Key；
5. 配置 DeepSeek API Key；
6. 配置 Ollama 本地地址；
7. 配置默认输出语言；
8. 配置默认改写强度；
9. 测试模型连接；
10. 保存配置到后端数据库。

页面结构：

```text
AI 设置中心

默认模型：
OpenAI / Gemini / Qwen / DeepSeek / Ollama / Local

API 配置：
OpenAI API Key
OpenAI Base URL
OpenAI Model

Qwen API Key
Qwen Base URL
Qwen Model

Gemini API Key
Gemini Model

Ollama Base URL
Ollama Model

按钮：
测试连接
保存设置
```

这样 AI 设置页不再空，也不会和内容创作页重复。

## 四、素材上传尺寸与大小提示

### 当前问题

素材上传模块缺少清晰提示，用户不知道应该上传什么尺寸、什么格式、文件多大以内。

### 修复目标

在素材上传区域明确提示平台推荐规格。

### 图片上传提示

```text
图片上传
支持 jpg / jpeg / png / webp
单张不超过 10MB
最多上传 9 张
推荐尺寸：
公众号配图：900×500 或 1080×608
知乎配图：1200×675
小红书图文：1242×1660 或 1080×1440
```

### 视频上传提示

```text
视频上传
支持 mp4 / mov / avi / mkv
单个不超过 500MB
最多上传 3 个
推荐尺寸：
B站：1920×1080，16:9
小红书：1080×1440，3:4 或 1080×1920，9:16
```

### 封面上传提示

```text
封面图上传
支持 jpg / jpeg / png / webp
不超过 10MB
推荐尺寸：
B站封面：1920×1080
公众号封面：900×383 或 900×500
小红书封面：1242×1660 或 1080×1440
```

### 前端展示建议

在上传模块中增加提示小字：

```text
支持 jpg/png/webp，单张 ≤10MB，最多 9 张
推荐：公众号 900×500；知乎 1200×675；小红书 1080×1440
```

对视频增加：

```text
支持 mp4/mov/avi/mkv，单个 ≤500MB，最多 3 个
推荐：B站 1920×1080；小红书 1080×1920
```

对封面增加：

```text
支持 jpg/png/webp，≤10MB
推荐：B站 1920×1080；小红书 1080×1440
```

## 五、建议修改左侧菜单结构

当前菜单：

```text
内容创作
平台预览
发布记录
账号管理
素材管理
AI 设置
帮助中心
```

建议保留，但调整功能定位：

| 菜单    | 新定位                  |
| ----- | -------------------- |
| 内容创作  | 当前任务的内容输入、素材选择、AI生成  |
| 平台预览  | 查看并编辑当前生成的多平台内容      |
| 发布记录  | 查看历史发布任务和帖子详情        |
| 账号管理  | 管理各平台账号授权状态          |
| 素材管理  | 管理历史上传素材库            |
| AI 设置 | 管理全局模型 API Key 和默认参数 |
| 帮助中心  | 展示使用教程、比赛说明、常见问题     |

不要把所有功能都堆在内容创作页。内容创作页只保留当前任务最关键功能，其他页面做全局管理。

## 六、推荐新增“工作台首页”

如果想让页面更完整，可以把默认首页改成“工作台”。

菜单结构改为：

```text
工作台
内容创作
平台预览
发布记录
账号管理
素材管理
AI 设置
帮助中心
```

工作台展示：

```text
今日发布统计
最近发布记录
素材库容量
AI 模型状态
平台账号状态
快捷入口：新建创作 / 查看记录 / 管理账号
```

这样系统打开后更像完整产品，而不是单一工具页面。

## 七、修复优先级

### 第一优先级

1. 接入后端数据库，解决刷新丢数据问题。
2. 发布记录支持查看详情。
3. 素材上传提示和校验。
4. AI 设置保存到后端。

### 第二优先级

1. 素材管理改成全局素材库。
2. 账号管理改成平台账号中心。
3. 发布记录详情页美化。
4. 新增工作台首页。

### 第三优先级

1. 素材使用次数统计。
2. AI 模型连接测试。
3. 发布失败重试。
4. 素材容量统计。

---

## 八、最终验收标准

修复后系统应满足：

1. 账户注册后，关闭前端重新打开仍然可以登录。
2. 用户信息保存在后端数据库中。
3. 发布记录保存在后端数据库中。
4. 点击发布记录可以查看完整帖子内容。
5. 素材上传后可以在素材管理页看到。
6. AI 设置保存后重新打开仍然存在。
7. 账号管理页能展示平台账号状态和授权信息。
8. 素材上传区域有明确尺寸、格式、大小提示。
9. 内容创作页不再堆叠过多全局管理功能。
10. 账户管理、素材管理、AI 设置页面都有独立价值。
